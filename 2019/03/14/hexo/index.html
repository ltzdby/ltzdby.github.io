如何部署和使用Hexo+GitHub创建个人博客

发表于 2019-03-14 更新于 2019-07-25

部署

GitHub设置

  1. 在GitHub上创建一个项目，项目名称必须为自己的GitHub用户名.github.io，即一个GitHub Pages仓库。
  2. 等10-30分钟，之后使用自己的GitHub用户名.github.io就能访问到这个GitHub Pages了。

安装Hexo

  1. 全局安装hexo-cli。

  2. cd进入Hexo源文件目录，初始化Hexo。

  3. 生成可以发布的静态HTML文件。

  4. 开启Hexo本地服务。

关联Hexo和GitHub Pages

  1. 生成SSH key。

    Linux系统下个人密钥默认在/home/用户名/.ssh目录下，Windows系统下个人密钥默认在C:/Users/用户名/.ssh下，macOS系统下个人密钥默认在/Users/用户名/.ssh下。如果没有公私钥可以在Linux下或macOS下用ssh-keygen命令生成。

  2. 上传公钥。

    将公钥.ssh/id_rsa.pub的内容复制，打开GitHub的个人设置-SSH and GPG keys-New SSH key，粘贴并保存。可在命令行用ssh -T git@github.com测试是否成功。

  3. 设置Git的username和email。

  4. 部署设置。打开Hexo源文件目录下的_config.yml，找到deploy部分。

    可在文章合适的地方加上。

  3. GitHub提供了自定义域名功能。

    • 首先，将你要转向的地址的域名解析CNAME记录指向你的用户名.github.io。
    • 然后，GitHub要求这个GitHub Pages仓库根目录下必须有一个文件名为CNAME的文件，内容为你要转向的域名。可以在Hexo源文件目录的source目录下新建这个文件，Hexo生成和部署时会直接将这个文件上传至仓库根目录。
    • 这样，访问用户名.github.io就会跳转到你指定的域名了。但实际上，访问的还是GitHub服务器上的这个GitHub Pages仓库。

  4. 将Hexo源文件目录也放到GitHub上进行版本控制。

    笔者就有惨痛的教训，手贱删除了原来的Hexo源文件目录，恢复无望以至于博客有一段时间就懒得再拾起来了。

    • 在Hexo基本部署完以后，在部署上去的文件所在的那个用户名.github.io仓库新建一个分支，比如叫hexo，并在仓库Settings里将其设置为默认分支。
    • 将这个仓库clone到本地，默认是一个用户名.github.io目录，进入这个目录，用git branch命令查一下，应该是在hexo分支上的。然后，把这个分支里的内容替换魏Hexo源文件目录内所有内容。
    • 执行git add .、git commit -m "xxx"、git push将其提交，这样，仓库的master分支就存储发布的静态内容，hexo分支就存储Hexo源文件目录的内容，再也不怕手贱删除Hexo源文件目录了。
    • 编辑、发布文章时，执行hexo d并将Hexo源文件目录下的内容提交一遍即可。有可能需要hexo clean。

使用Hexo的Next主题

安装Next主题

  1. 使用git克隆最新版本

  2. 或者：前往Next主题的GitHub release页面，选择需要的版本的Source code下载到本地，并解压到Hexo源文件目录的themes目录下，并将目录改名为next。

  3. 但是，对于我们把Hexo源文件目录也放到GitHub上托管的策略来讲，上述方法1是不可行的，因为git不允许一个仓库下还有一个仓库；方法2虽然可行，但是没办法与Next上游同步更新，对于一些小bug修复和功能增加、改进无法及时跟进。这里我采用了git的submodule处理一下。

    由于submodule仅仅是在仓库里创建一个指向submodule的链接，而使用Next主题需要对主题目录下的_config.yml配置文件进行修改，我们总不能把Next开源项目仓库里的配置文件给修改了吧（而且我们也没有权限这么做），所以我们需要先fork一下Next主题到自己的GitHub账户下，然后通过git submodule add https://github.com/用户名/hexo-theme-next themes/next将Next主题作为子模块添加到Hexo里。修改主题配置后，需要进入主题目录（也就是子模块里），执行git add .、git commit -m "xxx"、git push，然后再到Hexo下进行git操作。

    这时与Next新版本同步就很简单了，将fork的仓库与Next官方仓库同步即可。

文章中插入图片

绝对路径方法

当Hexo中只需要插入少量图片时，可以将图片统一放在source/images目录下，然后通过markdown语法访问它们，引用链接即为![](/images/image.jpg)。用这种方法，图片既可以在首页中被访问到，也可以在文章正文中被访问到。

相对路径方法

图片除了可以放在统一的目录中，也可以放在文章自己的目录中。文章的目录可以通过配置博客根目录下的_config.yml来生成。

在将post_asset_folder修改为true后，执行命令hexo n "文章文件名"时，在source/_posts/下就会生成文章文件名.md和同名的目录。将图片放在同名目录下，然后可以在文章中用![](image.jpg)访问图片。

上述引用方式，图片只能在文章正文中显示，不能在首页中正常显示。原因是image.jpg并不在网站根目录下，而是在文章文件名同名的目录下，只有在查看正文时，路径才是文章文件名同名目录的路径。
